# Audit Report — 2026-04-24

This audit reviewed core backend API handling, automated tests, and top-level
project documentation.

## What was checked

- Backend API behavior in `backend/app.py`
- Existing backend tests in `backend/tests/`
- Frontend unit tests (`jest`)
- Public docs consistency in `README.md` and `docs/API.md`

## Findings and fixes applied

### 1) Input validation gaps in API endpoints (fixed)

**Issue**
- `min_size` and `depth` were cast with direct `int(...)` calls in some routes,
  which could raise unhandled exceptions for invalid input (for example `0`,
  negative numbers, or non-integers) and produce HTTP 500 instead of a client
  error.

**Fix**
- Added centralized integer coercion with configurable minimum constraints in
  `backend/app.py`.
- Added explicit HTTP 400 responses for invalid `min_size` and `depth` while
  preserving `0` as a valid value.

### 2) Scan start path ergonomics inconsistency (fixed)

**Issue**
- `/api/duplicates` accepted both `path` (string) and `paths` (list), but
  `/api/scan/start` only reliably used `paths`.

**Fix**
- Updated `/api/scan/start` to accept `path` as well as `paths`.
- Added validation that `paths` must be a list or a string with non-empty
  string entries.

### 3) API documentation drift (fixed)

**Issue**
- `docs/API.md` was missing several implemented endpoints and validation
  behaviors.

**Fix**
- Documented missing endpoints:
  - `/api/organise/remove-preview`
  - `/api/preferences` (GET/POST)
  - `/api/scan_index/*`
  - `/api/maintenance/*`
- Added validation notes and `path`/`paths` behavior for scan start.

### 4) README release link + structure cleanliness (fixed)

**Issue**
- README pointed to a specific old release tag and had duplicated
  documentation sectioning.

**Fix**
- Updated release link to the releases listing page.
- Removed duplicated documentation block.

## Proposed next improvements (not implemented in this patch)

1. Generate API docs from route metadata (or OpenAPI) to reduce doc drift.
2. Add schema validation (e.g., Marshmallow/Pydantic) for request payloads.
3. Add CI checks for docs freshness (route list vs docs assertion).
4. Reduce broad exception handling in hot API paths and improve structured
   error logging for operator observability.
5. Add contract tests for optional endpoints (`scan_index`, maintenance mode)
   under feature-enabled fixtures.
